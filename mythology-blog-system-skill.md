## 二、_index.md 规范

### 2.1 根目录 `content/_index.md`

首页，简短开场白，文言文风格。Frontmatter 仅需 title。

```yaml
---
title: ""
---
```

### 2.2 博客列表 `content/posts/_index.md`

列出所有分类的链接，type 设为 "posts"。需随新体系更新。

```yaml
---
title: "博客"
type: "posts"
---
```

### 2.3 子目录 `_index.md`（分类 cascade）

使用 Hugo cascade 自动为目录下所有文章添加 categories。目前仅 `印度神话/` 目录有此文件。

```yaml
---
title: "印度神话"
cascade:
  categories:
    - "印度神话"
---
```

**规则：** 每个神话体系子目录可选择性添加 `_index.md`，添加后该目录下的所有 md 文件会自动继承 `categories` 字段。

### 2.4 分类页 `content/categories/_index.md`

Hugo taxonomy 页面，需列出所有分类链接。

### 2.5 标签页 `content/tags/_index.md`

## 三、文章 Frontmatter 规范

### 3.1 字段列表

```yaml
---
title: "文章标题"              # 必填。子神需标注父神关系："宙斯之子阿瑞斯""
url: /posts/greek/宙斯/       # 必填。推荐 /posts/[category]/[name]/
pageViewsKey: /posts/greek/宙斯/  # 必填。必须与 url 完全一致
date: 2026-05-03T07:23:49+08:00   # 必填。使用 current_time - 1h
draft: false                     # 必填。发布状态
categories:                      # 必填。用于 Hugo taxonomy 页面
  - 希腊神话
tags:                            # 必填。使用 YAML 列表格式（- 前缀）
  - 宙斯
  - 希腊神话
  - 众神之王
---
```

### 3.2 关键规则

- **子神 title 必须标注父神关系：** "达格达之子安格斯"、"埃尔之妻亚舍拉"、"宙斯之子阿波罗"
- **夫妻关系 title 标注：** "达格达之妻布里吉德"、"埃尔之妻亚舍拉"
- **tags 必须使用 `- 值` 列表格式**，禁止使用 `tags: ["值"]` 行内数组格式
- **pageViewsKey 必须与 url 完全一致**
- **categories 不可省略**（仅当子目录有 _index.md cascade 时可用）

## 四、文章正文结构规范

### 4.1 层级结构

```
# 标题：主标题（H1，不加编号）

<strong>关键词</strong>：关键词1、关键词2、关键词3、关键词4、关键词5、关键词6、关键词7、关键词8、关键词9、关键词10

---

## 前言

（引入主题，抛出核心问题或误解，约200-300字）

<!--more-->

---

## 一、大节标题

### 1.1 小节标题

（正文）

---

## 二、大节标题

...

---

## 结论

（总结全文，升华主题）

---

*（全文约XXXX字）*

&nbsp;

&nbsp;

---
```

### 4.2 格式细则

1. **标题层：** 仅一个 `#` H1 标题，不加编号。`##` 为二级标题，使用中文数字编号：`一、` `二、` `三、`
2. **关键词层：** 紧接标题后，用 `<strong>关键词</strong>：` 引导，后跟关键词列表，用中文顿号 `、` 分隔
3. **正文段落：** 段落之间空行，段落内不换行。用 `---` 作为视觉分隔（前言与正文之间、各大节之间）
4. **引用：** 使用中文引号 `""` 括起，不使用代码块
5. **强调：** 使用 `<strong>` 加粗，不使用下划线
6. **标点：** 全角中文标点（，。、：！？""''——）
7. **摘要：** `<!--more-->` 放在前言之后，用于生成首页摘要

### 4.3 文章类型模板

**类型 A：框架总述（每体系 1 篇）**
- 内容：自然地理背景、核心文献来源、神系谱系、宇宙观、与印欧神话的比较、现代影响
- 结构：前言 → 宇宙起源 → 神族体系 → 重要神祇 → 核心叙事 → 与印欧比较 → 结论

**类型 B：主神（每体系 1 位神王）**
- 内容：神格身份、形象与象征、神话事迹、家族谱系、与印欧神王的对比
- 标题：不加父子关系（本身就是父）
- 子标题示例："宙斯：众神之王与雷霆的权柄"、"奥丁：众神之王与智慧之神"

**类型 C：主神之妻（配偶神）**
- title 标注 "XX之妻"
- 内容：与主神的关系、自身神格、象征意义、神话故事
- 示例："朱庇特与朱诺：罗马的神圣王权"（夫妇合篇）、"布里吉德：达格达之妻"（独篇）

**类型 D：子神（主神之子女）**
- title 标注 "XX之子" 或 "XX之女"
- 内容：出身（父母）、神格与象征、主要神话、与印欧对应神比较
- 示例："安格斯：达格达之子"、"巴力：埃尔之子"

**类型 E：独立神祇**
- 无明确亲子关系，或关系不凸显
- 内容：神格起源、象征、神话事迹、跨文化比较
- 示例："塞尔努诺斯：鹿角之神"、"科塔尔：工匠之神"

## 六、格式注意事项（Pitfalls）

| # | 常见错误 | 正确做法 |
|---|---------|---------|
| 1 | title 缺失父子关系 | "宙斯之子阿瑞斯" 而非 "阿瑞斯" |
| 2 | tags 使用行内数组 `["a","b"]` | 使用 `- a` `- b` 列表格式 |
| 3 | pageViewsKey 与 url 不一致 | 两者必须完全一致 |
| 4 | 缺少 categories | 必须添加 categories 字段 |
| 5 | date 使用当前时间 | 使用 current_time - 1h |
| 6 | 关键词用 `**` 而非 `<strong>` | 使用 `<strong>关键词</strong>` |
| 7 | 缺少 `<!--more-->` | 前言后必须添加 |
| 8 | 使用英文逗号分隔关键词 | 使用中文顿号 `、` |
| 9 | 正文段落内换行 | 段落内不换行，段落间空行 |
| 10 | 引用使用代码块 | 使用中文引号 `""` |

## 七、创作流程

1. **新建体系：** 在 `content/posts/` 下新建子目录，添加 `框架总述.md`
2. **主神文章：** 写作神王（父神）的主神篇
3. **配偶篇：** 写作 "XX之妻" 篇
4. **子女篇：** 按谱系逐一写作 "XX之子/女" 篇
5. **更新列表：**
   - 更新 `posts/_index.md` 添加分类链接
   - 更新 `categories/_index.md` 添加分类页链接
   - 可选：添加 `_index.md`（cascade 自动赋 categories）
6. **验证：** 检查每个文件的 frontmatter 完整性
